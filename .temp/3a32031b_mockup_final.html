// Section 08
Implementation Phases
Phased deployment from foundation to full operational capability
Phase 1 // Foundation
Cross-Cutting Monitoring Agents
Deploy core monitoring infrastructure: health-monitor-001 (30s polling), db-monitor-001 (hourly + nightly maintenance), sec-monitor-001 (15-min aggregation), compliance-001 (daily + monthly audits). FastAPI backend on port 8801, SQLite for C&C state, asyncpg read-only connections to ClaimsAI and PolicyCentral databases.
Phase 2 // Core Agents
ClaimsAI & PolicyCentral App-Specific Agents
Deploy app-specific agents: claimsai-workflow-001 (SLA monitoring, bottleneck detection), claimsai-webhook-001 (WALTER integration health), policycentral-prod-001 (InsuredHQ sync monitoring). HITL workflows enabled, approval routing to designated leads.
Phase 3 // Application Integration
WALTER, HAL, Artemis Onboarding
Extend monitoring to WALTER (broker operations, HAL integration health), HAL (8-agent meta-monitoring, pricing validation), and Artemis (underwriting workflows). Full cross-application correlation enabled.
Phase 4 // Advanced Operations
Cross-Agent Coordination & Analytics
Advanced cross-agent communication, predictive analytics, anomaly detection with ML models, automated reporting dashboards, MAF migration for Azure-native deployment. Full operational capability achieved.

Technology Stack
Current implementation and future migration path
FastAPI
Agent Orchestration
Python 3.11 async backend on port 8801. Hosts all agent endpoints and scheduling.
SQLite
C&C State Store
WAL mode for concurrent reads. Zero infrastructure overhead. Stores agent state, alerts, audit logs.
asyncpg
App DB Connections
Read-only connection pools to application PostgreSQL databases. Enforced at connection level.
APScheduler
Agent Scheduling
Interval and cron-based job scheduling. Health (30s), Security (15min), DB (hourly), Compliance (daily).
OpenTelemetry
Observability
Distributed tracing, metrics collection, and log correlation across all agents and applications.
Azure Monitor
Cloud Observability
Log Analytics workspace, Application Insights, metric alerts, and diagnostic settings.
